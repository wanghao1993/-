# 文章摘要

## 推荐标题

我把公众号写作做成了一个 Skill，第一次验收就没通过

## 一句话摘要

把选题、核验、写作、配图和质检做成 Codex Skill 后，第一稿因为实战证据不足、目录过时和系列承接缺失没有通过验收。

## 短导语

第一次运行看起来很顺利：文件齐全，图片也生成了。可按新版标准验收后，写作 Skill 发现自己的第一稿只有 2607 个汉字，文章目录已经过时，标题承诺的「完整实战」也没有足够证据。

## 增长任务

- 主任务：建立信任
- 内容支柱：AI / Agent 工作流实战
- 系列位置：写作 Skill 的创建与第一次验收
- 后续文章：发布 7 天后的真实数据复盘

## 核心观点

- Skill 的价值不是一次生成更多字，而是让重复标准真正进入流程。
- 真实的写作实战应该包含证据、失败和修改理由，不只展示产物。
- AI 适合负责资料整理、结构化交付和机械检查。
- 作者必须为真实经历、账号方向、表达取舍和最终结论负责。
- 做成万人号不是写进一句目标，而是持续记录点击、完读、分享和关注转化并迭代规则。

## 推荐转发句

我给 AI 写了一份公众号工作说明书，然后让它检查自己。第一次验收，它没通过。这篇记录了真实的失败原因，以及我们怎样把单篇写作流程升级成长期增长闭环。
